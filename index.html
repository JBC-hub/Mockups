⚡

Choix du mode dès l'accueil

L'opérateur choisit immédiatement entre scan HU ou emplacement. Zéro menu superflu, 1 tap pour démarrer.

📷

Scan-First partout

Scan HU, scan emplacement, scan article, scan code centre de coût : toutes les saisies ont un bouton scan visible.

🎯

Type de mouvement clair

Icône + libellé pour chaque type (CC, Scrap, Rebut, Autre). Pas de codes à mémoriser pour l'opérateur.

🔢

Quantité totale par défaut

La quantité disponible est pré-saisie. Si sortie partielle, l'opérateur modifie uniquement ce qui change.

🔍

Récap avant confirmation

Écran de synthèse obligatoire avant le POST : article, qté, mouvement, destination comptable. Évite les erreurs irréversibles.

✅

Confirmation double

Pour Scrap/Rebut : validation en 2 taps (intention + confirmation explicite) car opération irréversible.

🗺️ Les deux flows applicatifs
Deux points d'entrée, même logique de fin de parcours

📦 Flow A — Depuis une HU

  • Étape 1 — Scanner ou saisir le code HU
  • Étape 2 — Visualiser le contenu HU (articles, lots, qtés)
  • Étape 3 — Choisir le type de mouvement (CC / Scrap / Rebut / Autre)
  • Étape 4 — Saisir le paramètre (CC, motif de rebut...)
  • Étape 5 — Confirmer la quantité (totale ou partielle)
  • Étape 6 — Récapitulatif → Valider → POST

🗂️ Flow B — Depuis un emplacement

  • Étape 1 — Scanner ou saisir l'emplacement
  • Étape 2 — Voir le stock de l'emplacement → sélectionner l'article
  • Étape 3 — Choisir le type de mouvement
  • Étape 4 — Saisir le paramètre (CC, motif)
  • Étape 5 — Confirmer la quantité
  • Étape 6 — Récapitulatif → Valider → POST
ℹ️ Correspondance SAP EWM /SCWM/ADGI Cette app mobile remplace la transaction ADGI bureau pour les opérations terrain. Les types de mouvements proposés correspondent aux catégories paramétrées dans l'entrepôt : sortie centre de coût (ZCC), mise au rebut (ZRB), scrap qualité (ZSQ), perte/damage (ZDG). L'accès aux lots (batch), aux types de stock spéciaux et aux UGI peut être activé selon le paramétrage client.

Erreurs SAP claires : pas de code d'erreur technique brut — le message est traduit en termes métier avec une action corrective proposée directement depuis l'écran d'erreur.
Succès avec référence document : le numéro du document EWM créé est affiché immédiatement pour traçabilité.
